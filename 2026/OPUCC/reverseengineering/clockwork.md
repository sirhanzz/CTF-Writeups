<h2>Solution</h2>

<p>
This challenge provides a 64-bit ELF executable that checks whether the
user enters the correct key. Since the correct flag is not visible through
normal execution, the binary must be analyzed using reverse engineering.
</p>

<h3>Step 1: Inspect the Binary</h3>

<p>
Using tools such as <b>strings</b>, <b>Ghidra</b>, or <b>IDA</b>, several
useful symbols were found because the binary was not stripped.
</p>

<pre>
main
checkFlag
gear
gears
state
rotl8
clockwork.c
</pre>

<p>
The presence of these symbols makes it clear that the validation logic is
implemented inside the <b>checkFlag()</b> function.
</p>

<h3>Step 2: Analyze checkFlag()</h3>

<p>
The <b>checkFlag()</b> function reads the user's input and processes each
character individually. Instead of comparing the input directly with the
flag, each character is transformed before comparison.
</p>

<p>
The transformation uses three important components:
</p>

<ul>
    <li><b>gear()</b> – Generates a pseudo-random byte.</li>
    <li><b>rotl8()</b> – Performs an 8-bit left rotation.</li>
    <li><b>gears[]</b> – Stores the encrypted target bytes.</li>
</ul>

<p>
The program also initializes an internal state using the input length,
ensuring the same pseudo-random sequence is generated during validation.
</p>

<h3>Step 3: Validation Algorithm</h3>

<p>
The validation process can be simplified as follows:
</p>

<pre>
state = strlen(flag) ^ 0x1337C0DE;

for each character:
    g = gear()
    x = flag[i] XOR g
    x = ROTL8(x, g & 7)

    compare x with gears[i]
</pre>

<p>
If every transformed byte matches the encrypted values stored inside
<b>gears[]</b>, the program prints the success message.
Otherwise, it prints an error indicating the key is incorrect.
</p>

<h3>Step 4: Reverse the Algorithm</h3>

<p>
Since XOR and bit rotation are reversible operations, the algorithm can be
inverted.
</p>

<p>
For every encrypted byte stored in <b>gears[]</b>, the following operation
is applied:
</p>

<pre>
flag[i] = ROTR8(gears[i], g & 7) XOR g
</pre>

<p>
By reproducing the same pseudo-random sequence generated by
<b>gear()</b>, each encrypted byte can be decrypted to recover the
original flag.
</p>

<h2>Conclusion</h2>

<p>
This challenge demonstrates a common reverse engineering technique where
user input is transformed using bitwise operations and compared against an
encrypted byte array. By reversing the XOR and bit-rotation operations and
reproducing the same pseudo-random sequence, the original flag can be
successfully recovered.
</p>
